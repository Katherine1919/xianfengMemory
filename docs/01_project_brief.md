# 拾光源点·先锋社区记忆智能体项目简报

## 1. 项目名称

拾光源点·先锋社区记忆智能体。

## 2. 项目一句话定位

一个从 HTML 展示 Demo 起步的社区记忆智能体原型，用先锋社区的规划档案、历史线索、街巷点位、非遗美食和社区生活内容，生成可浏览、可提问、可演示的本地记忆入口。

来源：
- `先锋社区智能体项目提案.pdf`，第 1 页：定位为“从线下展厅互动屏起步的长期社区记忆系统”，并强调社区记忆、公民众包、本地体验入口。
- `拾光源点•先锋社区 1.0.pdf`，第 3-4 页：强调北宋石板桥、轮滑之乡、传统美食、先锋小学，以及儿童、乐龄、亲子、青年客群。
- `100个问题和答案.docx`，第 1-3 段、第 115-117 段：明确 HTML MVP 原型、展厅互动屏、多页面、真实 API、真实上传、管理员审核、手绘地图等第一版方向。

## 3. Demo 第一版目标

第一版是一个面向展厅互动屏的多页面 HTML MVP 原型，核心是“可体验、可演示、可迭代”。根据 `100个问题和答案.docx` 的已填答案，第一版目标不只是展示，还要为后续真实采集和真实智能体预留结构：

- 用真实资料支撑 Story Archive，不再写空泛的“岭南文化”“老街故事”。
- 把保护对象、街巷肌理、历史环境要素、非遗美食、社区客群整理成可写入 `js/data.js` 的结构化素材。
- 让用户可以通过手绘地图、故事卡片、时间线、关键词和智能体问答理解先锋社区。
- 支持真实上传方向：文字、照片、音频、视频、地理位置；最小记忆字段包括标题、讲述人、时间、地点、故事正文。
- 支持管理员审核方向：查看提交、编辑、删除、审核、加标签、导出数据。
- 呈现“展厅故事牌 + 社区口述 + 历史档案”的混合风格。
- 明确标注来源，区分官方规划资料、社区展陈资料、历史信息整理和后续可能采集的口述内容。
- 视觉方向为干净简洁、岭南风；展厅大屏优先。

## 4. 用户角色

### 居民与长者

- 角色：讲述者、校验者、社区记忆贡献者。
- 需求：被尊重地记录；不被篡改；能看到自己的社区被认真呈现。
- 来源：`先锋社区智能体项目提案.pdf`，第 1-2 页。

### 儿童与学生

- 角色：体验者、任务参与者、学校项目采集者。
- 需求：通过任务、地图、故事卡认识家门口的历史。
- 第一版儿童核心年龄段为 7-12 岁。来源：`100个问题和答案.docx`，第 115 段。
- 来源：`先锋社区智能体项目提案.pdf`，第 2、5 页；`拾光源点•先锋社区 1.0.pdf`，第 4、6、11-12 页。

### 青年与志愿者

- 角色：采集者、整理者、二次创作者。
- 需求：参与社区建设，获得项目经验、工作坊和志愿证明。
- 来源：`先锋社区智能体项目提案.pdf`，第 2、5 页；`拾光源点•先锋社区 1.0.pdf`，第 4、8、11、13 页。

### 亲子家庭

- 角色：浏览者、任务陪伴者、社区活动参与者。
- 需求：有地方可逛、有故事可讲、有轻松的亲子活动。
- 来源：`拾光源点•先锋社区 1.0.pdf`，第 4、12 页。

### 游客与外来参观者

- 角色：见证者、传播者、导览使用者。
- 需求：快速理解先锋社区为什么重要，看到可信、具体、有温度的本地内容。
- 来源：`先锋社区智能体项目提案.pdf`，第 2、6 页。

### 社区组织、学校、运营方

- 角色：管理者、审核者、活动组织者。
- 需求：有资料库、有故事种子、有演示路径、有后续扩展框架。
- 来源：`先锋社区智能体项目提案.pdf`，第 2-5 页。

### 社区管理员

- 角色：审核者、数据整理者、后台使用者。
- 需求：查看提交、编辑、删除、审核、加标签、导出数据；上传内容要先经管理员审核。
- 来源：`100个问题和答案.docx`，第 117 段。

## 5. 核心功能

第一版 HTML Demo 可围绕这些功能设计内容：

- Story Archive：展示故事卡片、点位、年代、来源、可信度。
- Memory Map：使用手绘地图方向，把蚝壳墙、古井、门楼、麻石巷、先锋小学、十八巷等转成不少于 10 个地点节点。
- Timeline：北宋石板桥线索、明清市镇、水陆贸易、民国骑楼、建国后先锋小学、当代老城市新活力。
- Ask Xianfeng：智能体名称可用“拾光仔”，角色为记录员、故事讲述者；说话风格亲切、儿童友好，可支持粤语词汇。
- Children Mission：面向儿童的寻宝任务、问答题、错题反馈、亲子任务、绘画/作文/录音提交；奖励方向为贴纸 + 积分 + 记忆通行证。
- Memory Upload：支持真实上传方向，字段至少包括标题、讲述人、时间、地点、故事正文，并可区分个人记忆、家庭记忆、社区事件、建筑记忆、美食记忆、学校记忆。
- Admin Review：上传内容需要管理员审核，可设置公开、仅社区可见、仅管理员可见、不公开仅存档。
- Source Notes：每条故事标注来源文件和页码，资料不足时显示“资料不足，需人工补充”。

## 6. 3 分钟演示路径

1. 开场 20 秒：进入首页，说明这是“拾光源点·先锋社区记忆智能体”展厅大屏 HTML MVP，不是政府汇报页。
2. 40 秒：打开 Story Archive，看 3 张代表卡片：蚝壳墙、先锋小学、前锋十八巷。
3. 40 秒：切换到地图或点位视图，展示古井、门楼、麻石巷、蚝壳墙等历史环境要素如何落点。
4. 40 秒：打开一个故事详情，展示“历史档案依据、空间细节、社区口述占位、儿童任务、来源说明”。
5. 30 秒：演示“拾光仔”问答，例如“先锋大街和前锋大街有什么关系？”回答必须引用知识库。
6. 30 秒：演示上传记忆和管理员审核入口，说明后续可升级为社区长期数据库、语音转文字、AR 导览、线下互动屏。

## 7. 后续升级路线

### 第一阶段：展厅大屏 HTML MVP

- 多页面小网站，页面包括首页、地图、时间轴、上传记忆、后台；故事库和智能体对话可作为核心模块合并或独立。
- 技术上允许使用 React.js，但当前资料建设阶段只为后续数据结构做准备。
- 重点是 Story Archive、手绘地图点位、时间线、上传记忆、管理员审核和智能体问答。
- 数据来自本次 `docs/` 知识库。
- 来源：`100个问题和答案.docx`，第 115-117 段。

### 第二阶段：可维护内容库

- 把故事、点位、关键词拆为 JSON 或 CMS 数据。
- 增加来源字段、可信度字段、资料缺口字段。
- 增加人工审核状态。

### 第三阶段：真实采集与审核

- 接入口述、图片、学校采集、居民上传。
- 建立同意、匿名、复核、版本记录机制。
- 来源：`先锋社区智能体项目提案.pdf`，第 2-5 页。

### 第四阶段：真实 AI 和检索

- 接真实 OpenAI API、数据库、搜索引擎、语音转文字、AI 摘要和问答。
- 对隐私、未成年人、争议记忆做分级权限。
- 来源：`先锋社区智能体项目提案.pdf`，第 3-4 页。

## 8. 项目边界

- 当前资料建设阶段只输出知识库文件，不改网站 UI，不重写 `js/data.js`。
- 第一版产品方向是 HTML MVP，可为真实 API、真实上传、管理员审核预留结构；但在本次资料任务里不实现这些功能。
- 当前所有“口述感”内容只能作为写作风格，不得冒充真实居民原话。
- 当前数据必须来自 `source_materials:/` 已读取资料或明确标注“资料不足，需人工补充”。

## 9. 网站不应该是什么

- 不是政府汇报页：不要只堆规划口号、指标和大标题。
- 不是普通旅游页：不要把先锋社区写成泛泛打卡景点。
- 不是空洞 AI 概念页：不要只写“智能、连接、赋能”，必须落到具体街巷、地址、物件、人物角色和来源。
- 不是没有来源的历史故事站：不能编北宋故事、名人故事、居民故事。
- 不是单纯商业招商页：拾光源点内容可以服务运营，但 Story Archive 应先服务社区记忆和可信档案。
