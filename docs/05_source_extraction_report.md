## 7. 后续如何写入 `js/data.js`

建议下一步按 5 类数据重写：

1. `knowledgeFacts`: 从 `02_xianfeng_knowledge_base.md` 抽取短事实，用于 AI 问答和来源展示。
2. `storySeeds`: 从 `03_story_seed_database.md` 转成故事卡片数组。
3. `mapPoints`: 从故事种子和 `100个问题和答案.docx` 的地点清单里抽取可定位点位，如蚝壳墙、古井、门楼、麻石巷、先锋小学、前锋十八巷、工会建筑群。
4. `timelineEvents`: 北宋石板桥线索、明清水陆贸易、民国骑楼、1925 文峰小学、1949 先锋小学、2000 轮滑凤舞、当代拾光源点；分段使用北宋、明清、民国、建国初期、改革开放、当代。
5. `contentRules`: 把 `04_ui_content_rules.md` 中的 sourceNote、credibility、资料不足、上传审核和隐私规则转成 UI 展示逻辑。
6. `uploadSchema`: 根据 `100个问题和答案.docx` 建字段：标题、讲述人、讲述人年龄段、时间、地点、故事正文、内容类型、上传媒体、公开范围、授权同意、审核状态。
7. `adminQueue`: 预留查看提交、编辑、删除、审核、加标签、导出数据的后台数据结构。

建议先做数据结构，不先改视觉：

- 每条故事必须有 `sourceNote`。
- `credibility: low` 的卡片显示“资料不足”角标。
- `credibility: conflict` 的卡片显示“需核对”角标。
- 没有真实图片的卡片使用 `suggestedImageStyle` 生成 CSS 示意封面。
- 所有居民口述内容先用 `oralHistoryQuestions`，不要写假口述。
- 上传内容默认进入 `pendingReview`，不要直接公开。
